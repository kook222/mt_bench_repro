# 향후 연구 방향

현재 재현 실험 결과를 바탕으로 도출한 연구 아이디어 및 분석.

---

## Inconsistency 46% — Position Bias 분석

### 무엇인가

Pairwise judge는 동일 질문에 대해 **두 번** 실행한다:

- **AB 순서**: "A 답변 vs B 답변"을 judge에게 제시 → 판정
- **BA 순서**: "B 답변 vs A 답변"을 judge에게 제시 → 판정

두 결과가 **일치하면** winner 선언, **불일치하면** `inconsistent` (집계 시 tie 처리).

이는 논문 Section 3.4의 **conservative approach** — position bias(먼저 제시된 모델을 선호하는 경향)를 완화하기 위한 설계다.

### 이번 실험 결과

| 항목 | 수치 |
|------|------|
| 전체 pairwise 판정 | 1,200개 (15쌍 × 80문항) |
| inconsistent | 553개 |
| **inconsistency율** | **46.1%** |

즉 Qwen2.5-14B judge는 **절반 가까운 비율**에서 AB 순서와 BA 순서의 판정이 달랐다.

### 왜 높은가

논문은 GPT-4를 judge로 사용했으며 inconsistency율이 낮았다. Qwen2.5-14B는 GPT-4 대비:
- 모델 크기가 작아 instruction following 능력이 부족
- Position bias에 더 취약 — 먼저 나온 답변을 선호하는 경향이 강함
- 결과적으로 AB/BA 순서만 바꿔도 판정이 자주 뒤집힘

### 함의

inconsistency율이 높을수록:
- Pairwise 결과의 신뢰도 하락
- 유효 게임 수 감소 (1200개 → 647개만 집계)
- 모델 간 미세한 성능 차이를 포착하지 못함

이번 실험에서 single-answer 순위와 pairwise 순위가 일치한 것은, inconsistency 항목이 모두 tie로 처리돼 순위에 큰 영향을 주지 않았기 때문이기도 하다.

## 향후 연구 아이디어

### 1순위: Judge 모델 크기 스케일링 실험

**연구 질문:** Judge 모델 크기가 커질수록 inconsistency율이 어떻게 감소하는가? 신뢰할 수 있는 판정을 위한 최소 judge 크기는 어디인가?

**실험 설계:**

| Judge 모델 | 파라미터 | 예상 inconsistency | 비용 |
|-----------|---------|-------------------|------|
| Qwen2.5-7B | 7B | ~60% 이상 예상 | 무료 (서버에 있음) |
| Qwen2.5-14B | 14B | **46.1% (측정됨)** | 무료 |
| GPT-4o-mini | — | ~20% 예상 | API 소액 |
| GPT-4o | — | ~10% 예상 | API |

**기여점:** "Judge Scaling Law" — judge 크기와 inconsistency율의 관계를 실증적으로 도출. 어느 크기부터 GPT-4급 신뢰도에 수렴하는지 보여주는 곡선.

**실행 방법:**
```bash
# Qwen2.5-7B judge (이미 서버에 있음)
JUDGE_MODEL_ID=Qwen2.5-7B-Instruct bash scripts/run_judge_multi_a100.sh

# GPT-4o-mini judge
JUDGE_USE_VLLM=false OPENAI_API_KEY=... bash scripts/run_judge_multi_a100.sh
```

### 2순위: 한국어 MT-Bench — 언어별 모델 서열 변화

**연구 질문:** 영어 기준 MT-Bench 순위가 한국어 평가에서도 동일하게 유지되는가?

**실험 설계:**
- 80문항 전체를 한국어로 번역 (DeepL 또는 GPT-4o 활용)
- 동일 7개 모델로 답변 생성
- 동일 judge 파이프라인 적용
- 영어 순위와 한국어 순위 비교

**예상 흥미로운 결과:**

| 모델 | 영어 순위 | 한국어 예상 | 이유 |
|------|----------|------------|------|
| Qwen2.5-7B | 1위 | 유지 or 상승 | 중국어/아시아 언어 강점 |
| Yi-1.5-9B | 4위 | 상승 가능 | 다국어 훈련 데이터 |
| Mistral-7B | 5위 | 하락 가능 | 유럽어 중심 훈련 |
| Zephyr-7B | 7위 | 하락 가능 | 영어 중심 fine-tuning |

**기여점:** 영어 벤치마크 순위를 다국어 상황에 그대로 적용하는 것의 위험성 실증. 한국어 LLM 선택 가이드라인 제시.

### 3순위: 문항 수(Question Size) 민감도 분석

**연구 질문:** 신뢰할 수 있는 모델 순위를 얻으려면 MT-Bench 80문항 중 몇 문항이 필요한가?

**실험 설계:**
- 80문항 중 무작위 서브샘플링: 10 / 20 / 40 / 60 / 80문항
- 각 크기에서 Spearman 순위 상관계수 계산 (80문항 전체 결과 대비)
- 카테고리별 분석 (math는 10개로도 수렴? writing은 더 많이 필요?)

**기여점:** MT-Bench 경량화 가능성 탐색. 평가 비용을 줄이면서도 신뢰도를 유지하는 최소 문항 수 제안.

---

### 4순위: 변별도 기반 새 갭 지표

**연구 질문:** Hard/Easy를 카테고리 고정이 아닌 "모델 간 점수 분산"으로 정의하면 무엇이 보이는가?

**아이디어:**
- 현재: hard = math/reasoning/coding (카테고리 기반 고정 분류)
- 제안: 각 문항에서 7개 모델 점수의 표준편차 → 분산 높은 문항 = 실제로 어려운 문항
- 분산 기반으로 문항을 재분류하면 카테고리와 일치하는지, 아니면 다른 패턴이 나오는지 분석

**기여점:** 벤치마크 문항의 변별도를 데이터 기반으로 측정하는 방법론 제안.

## 추천 실행 순서

```
1. Judge 크기 실험 (Qwen2.5-7B 추가, GPT-4o-mini 추가)
   → 이미 인프라 구축 완료, 추가 실험 2개만 하면 됨
   → 기존 46.1% 데이터와 합쳐 즉시 논문 가능

2. 한국어 번역 실험
   → 번역은 GPT-4o API로 하루 안에 처리 가능
   → 한국 연구자로서 motivation이 명확해 심사 설득력 높음

3. 1 + 2를 합친 2×N 실험
   → Judge 크기 × 언어(영어/한국어) 조합
   → 가장 논문다운 구성 (2개 축으로 ablation 가능)
```
